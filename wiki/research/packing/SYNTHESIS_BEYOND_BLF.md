- The existing boundary fixture is too easy (part-area/stock approx 0.87, ceiling approx 86.9%): the 84.74% base leaves only approx 2 points of headroom, so the GLS lever cannot be demonstrated on it. A harder oversub-holed fixture MUST be built and the base measured on it before any multi-point GLS claim. Without this the lever's value is unproven.
- No apples-to-apples gap-to-sparrow exists: util_stock (true-area / sheet-minus-holes) is not comparable to sparrow's bbox-density (piece-area / bounding-box). Until >=1 ESICUP instance is run, every SOTA-parity statement is unsupported (AGENTS.md sec9).
- GLS runtime: at K approx 1e3-1e4 descent steps it is far slower than the current 5590ms uncached single BL pass. The NFP cache + AABB-grid (net48, in scope) are REQUIRED, not optional, or the lever is impractical on 40-60 part fixtures within reasonable time.
- Stage C seeding API gap: IrregularSheetFillNfpBlf is sealed, PlaceAll is private, no ctor accepts pre-placed obstacles. The cheap path (ring loops as extra sheetHoles) must be verified to actually exclude those regions; the clean path (new public obstacle-seeding entry) is real engine work. Do not state 'the only wiring is invoking PlaceAll'.
- V506 spacing floor (Math.Max(0.1,spacing), line 141) vs NFP-BLF floor (Math.Max(0.0,spacing), line 93): Stage C must call NFP-BLF directly at spacing 0, else the 0.1 floor caps interior density below the 84.74% bare-C bar.
- Stage-C feasible-region fragmentation: ring obstacles fixed before interior fill can split the feasible region and DROP total util_stock below bare-C. The regression guard (A/B kept only if total >= bare-C) is mandatory; without it the bmode3 evolution can regress.
- True concave IFP erosion is load-bearing once GLS slides parts to the boundary; shipping GLS on concave sheets WITHOUT it risks containment violations the hull-vertex IFP over-admits.
- Determinism: SortParts uses _seed==0?null (no RNG at the default seed). Any SA/GRASP/GLS driver must carry its own nonzero-seeded System.Random or the byte-identical-trajectory contract breaks; add a same-seed-twice assertion.
- HITL gates: a commit touching > 5 files needs HITL (AGENTS.md sec6); any Validator.cs fixture/metric change is a harness edit; no util_stock or density number is project-truth until Libish visually validates the .3dm in Rhino (truth criterion c). All numbers here are code-measured or literature-estimated, not visually validated.
- net48 hygiene on all new files: #nullable disable per file, no string.Contains(StringComparison), no HashCode.Combine; Core (OverlapMinNester, true-IFP helpers, cache) must stay Rhino-free (Clipper2Adapter only), do not pull RhinoCommon into the GLS math.
