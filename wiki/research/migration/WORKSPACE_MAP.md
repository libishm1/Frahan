## directoryConvention

AGENTS.md "core memory model" defines five convention names: wiki / raw / outputs / data / research. Realization is PARTIAL:

- raw/ = immutable evidence, ADD-ONLY, never edit (header required). NO root-level raw/ exists; the active evidence store is Template-General/raw/ (13.8GB). Datasets live ONLY under Template-General/raw/<YYYY-MM-DD>/.
- wiki/ = curated, owned project memory, edits HITL-gated. The LIVE one is root D:/code_ws/wiki/ (~312 files). Subdirs: algorithms/, specs/ (00-22 + named), papers/ (+equations_and_diagrams/), research/ (slm_cards/, slm_spines/, roses_synthesis/, +long-form), index/ (cross-cutting audits incl data_assets_inventory.md), testing/, fabrication_workflows/, agent_prompts/. Top files: index.md, log.md, health.md, audit_trail.jsonl.
- outputs/ = generated, date-stamped YYYY-MM-DD/<task>/. Root outputs/ (~1.23GB) spans 2026-04-30..2026-06-06 + releases/. Template-General/outputs/ (12.1GB) holds the plugin + all HITL cards.
- data/ = the convention RESERVES this name but NO data/ or Data/ dir exists anywhere yet. Task #42 ("D-drive dataset hunt -> code_ws/Data") is the plan to create it. Until then datasets sit under Template-General/raw/.
- research/ = NOT a top-level dir; it lives as wiki/research/. Separately, a root-level reference/ (~3.1-7.6GB, gitignored) holds non-wiki precedent assets (papers, Quarra video, mrac_workshop, design-system zips) — these are design/precedent references, NOT pipeline inputs.
- checkpoints/ = immutable state archives paired with git tags (root, ~10-12 entries; dated + named e.g. frahan-v1.0-rc1, trencadis-pipeline-hitl-1).

Date-stamp rule: every generated artifact goes under outputs/YYYY-MM-DD/<task-slug>/. raw is add-only with a provenance header (_SOURCE.md per dataset dir). wiki is the only "edit" surface and only with HITL approval.

Root cause: quadric edge-collapse pathologically slow at multi-million scale. Fix: never Mesh.Reduce inline on multi-million meshes; use O(n) voxel-cluster decimation via bulk arrays (~15-20s for 2.2M). Transforms/bakes on multi-million are fine; only Reduce is the trap.

Code fix APPLIED (commit 7d60674) but HILT 3dm/gh/png QUEUED. User APPROVED pending intentionality test.
