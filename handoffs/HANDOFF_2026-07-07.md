## Browser nest demo — LIVE

<https://libishm1.github.io/Frahan/nest/>. Client-side WASM, no backend
(the `ContactNfpHoleNester` engine compiled to WebAssembly; CAD file never
leaves the browser). Built in CI by the docs workflow (resilient:
continue-on-error, so a WASM hiccup never breaks the docs deploy).

Hard-won gotchas already fixed (do not reintroduce):
- Do NOT call `dotnet.run()` in main.js — it exits the runtime; use create() +
  getAssemblyExports (runtime stays resident).
- WASM trims reflection: JSON must be source-generated (`JsonSerializerContext`),
  not reflection-based.
- Eager load, not lazy: lazy racing the ready-event left Nest grayed
  ("doesn't load at all"); main.js eager-creates + dispatches `frahan-ready`,
  app.js enables Nest race-proof (checks `globalThis.frahan` OR the event).
- CI: app + engine must be the SAME TFM (net9); mismatched TFMs fail the WASM
  build. `dotnet workload restore` in CI.

Features: DXF/SVG import (red shapes = sheet defects, others = parts), sample
with a "with defects" toggle, boundary-hug, SVG export.
